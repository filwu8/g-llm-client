# 本地文件任务 V1

## 本轮范围

G-LLM 可以对通过系统文件选择器添加的本地附件建立受控任务，识别“最多 2097152”“不超过 2 MB”等大小限制，先展示计划，经用户确认后处理并验证输出。

当前自动执行范围：

- JPG、JPEG、PNG、WebP、GIF：超过限制时转为 JPEG 并迭代调整质量和分辨率。
- PDF：跨平台渲染页面并重建 PDF，在 3600 至 640 像素的多档分辨率中对 JPEG 质量执行二分搜索，比较所有满足上限的真实候选并选择画质评分最高的一份；执行前提示文本搜索、链接、表单和数字签名可能丢失。
- 已经小于限制的文件：复制到结果目录并验证，不修改源文件。
- 其他超过限制的文件：明确标为暂不支持，不执行破坏性转换，不报告虚假成功。

输出默认保存到第一个源文件旁的 `处理后附件` 文件夹。同名文件不会覆盖，而是生成带序号的新文件。

## 安全边界

- 本地路径只保存在 Electron 主进程的附件登记表中，模型上下文不包含绝对路径。
- Renderer 只能提交附件 ID，不能指定任意输入路径。
- 打开输出目录时使用任务 ID 反查主进程登记的目录，不能让 Renderer 打开任意路径。
- 不开放 Shell、脚本执行、删除或覆盖源文件。
- 每个输出文件完成后重新读取实际字节数并验证。
- 文件大小下限只作为真实画质候选的择优条件。严禁通过追加空白、随机或无意义字节伪造文件大小；PDF 二进制压缩不得绕过专用工具改用通用脚本覆写。

## 验收步骤

1. 选择三张大于 2 MiB 的本地图片。
2. 输入：`下面三个文件太大，上传提示附件超出上传限制，最多上传2097152，你帮我弄一下。`
3. 确认计划显示目标为 `2,097,152 字节（2 MiB）`。
4. 点击“开始处理”。
5. 确认每个成功文件的处理后大小均不超过 2,097,152 字节。
6. 点击“打开输出文件夹”，检查输出可正常打开、源文件未变化。
7. 加入一个超过限制的 PDF，确认界面提示页面重建风险；处理后检查页数、清晰度和文件大小。

- DOCX 内嵌图片压缩。
- 固定工作区与目录授权。
- 任务取消、持久化、恢复和模板。
- 更通用的模型工具调用与多步 Agent 循环。
