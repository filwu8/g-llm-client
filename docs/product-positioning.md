# G-LLM Product Positioning

这份文档用于对外文案、官网、下载页和销售材料，避免把 G-LLM 只描述成普通聊天软件。

## 一句话定位

G-LLM 是面向 Windows、macOS 和 Linux 的跨平台桌面 AI 工作台，支持用户接入自己的模型供应商，并把助手、知识、截图、文件和联网搜索放到同一个本地优先的工作流里。

## 核心卖点

- 跨平台：同时支持 Windows、macOS、Linux，不把用户锁定在单一系统。
- 模型开放：支持 G-LLM 网关和 OpenAI-compatible 供应商，用户可以选择自己的 API Key、模型和网关。
- 本地优先：会话、助手、知识库、记忆和配置优先保存在本机。
- 助手优先：用户围绕场景创建助手，而不是每次从空白聊天开始。
- 桌面原生：Windows 支持托盘驻留、悬浮 logo、快速对话、截图提问和单进程保护。
- 资料输入丰富：支持文件、图片、截图、剪贴板粘贴、知识库和联网搜索。
- 商业可控：社区版 AGPL，商业闭源分发、白标、企业部署可通过商业授权处理。

## 和同类产品的差异化

对标 WorkBuddy 等桌面 AI 助手时，G-LLM 的表达重点应放在“开放、跨平台、本地优先、可商业定制”，不要只强调聊天能力。

- 如果对方强调 AI 助手效率，G-LLM 强调多助手、多供应商、多模型和用户自有 API Key。
- 如果对方强调 Windows 桌面体验，G-LLM 强调 Windows/macOS/Linux 三端一致推进，同时保留 Windows 托盘和悬浮小窗体验。
- 如果对方强调团队或企业，G-LLM 强调品牌自有、网关可控、商业授权、白标和企业部署空间。
- 如果用户担心隐私，G-LLM 强调本地数据和匿名统计边界，不采集聊天内容、API Key、文件内容、截图内容、知识库内容或记忆内容。

## 当前最该补齐的短板

- 可信分发：Windows 代码签名、Microsoft Store/MSIX、macOS 签名公证。
- 稳定性：三端发布前 QA 清单、启动日志、闪退反馈路径。
- 官网与 README：清楚展示三端下载、核心能力、版本号、安装提示和隐私边界。
- 任务型能力：后续可以增强更完整的任务计划、执行、回放和结果归档能力。

## 推荐对外文案

短版：

```text
G-LLM 是一款跨平台桌面 AI 工作台，支持 Windows、macOS 和 Linux。你可以接入自己的模型供应商，创建场景助手，并在同一个应用里完成聊天、文件理解、截图提问、知识引用和联网搜索。
```

更强调企业版：

```text
G-LLM 为企业和团队提供可控的桌面 AI 客户端能力，支持 OpenAI-compatible 模型接入、本地优先的数据管理、助手工作流、知识与文件输入，以及商业授权、白标和私有化部署合作。
```
