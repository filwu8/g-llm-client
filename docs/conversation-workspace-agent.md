# 会话工作区 Agent

会话可以单独绑定一个本地目录。未绑定目录时保持普通聊天；绑定后，同一输入框切换为工作区 Agent，并复用当前会话选择的模型供应商和模型。

## 当前工具

- 查看目录、检查文件、读取 UTF-8 文本文件。
- 提取 PDF、Word（DOCX）和 PowerPoint（PPTX）正文，用于文档分析。
- 按文件名或文本内容搜索。
- 创建目录和文本文件。
- 精确替换文件内容。
- 移动或重命名文件。
- 压缩图片和 PDF 到指定字节数。
- 当专用工具不足时，自主生成并运行临时 JavaScript，处理批量文件、文本、JSON、CSV 和代码任务。
- 通过当前供应商的图片生成接口生成 PNG 并保存到工作区。

## 自主执行循环

工作区 Agent 优先选择可靠的专用工具。没有匹配工具时，模型可以调用 `run_javascript`，通过受控 `workspace` API 查看、读取、写入、复制和移动授权目录内的文件。执行器自动识别脚本产生或修改的文件，并要求模型在最终回复前再次检查产物；检查失败时，模型可以修正代码后重试。

目录授权保存在会话本身，后续消息和失败重试都会继续使用同一目录，直到用户主动解除授权。新建会话不会继承上一个会话的目录。

当会话超过 32 条消息或约 48,000 字符时，较早历史会自动压缩为时间线摘要，最近 24 条消息保留原文。摘要同时保留已生成文件和关键工作区操作，使长会话压缩后仍能延续项目状态。

工作区 Agent 的多轮请求同时保留近期图片附件、可提取的附件正文、知识引用、工具执行结果和生成文件。最多携带最近 4 张图片且总计不超过 12 MB。工作目录状态作为系统背景提供，真正的最新用户消息始终位于请求末尾；用户补充图片或资料时不会被目录观察消息覆盖。

首次进入工作区或用户明确提到目录、文件和修改任务时会同步目录。普通追问、评价或图片补充沿用上一轮工作区上下文，不会无条件重复读取目录。除非用户要求创建、修改或保存文件，Agent 默认直接回答，不主动生成额外文件。

## 会话项目长期记忆

普通会话和工作区会话共享同一套项目长期记忆。会话至少完成两轮后建立首份记忆，之后每累计约三轮新对话后台更新一次，以减少额外模型请求。记忆使用当前会话选择的供应商和模型，不增加新的 API Key。

项目记忆结构化保存项目概况、已确认需求、已确认决策、业务规则、关键对象、待确认事项和风险。提炼提示明确禁止把助手建议当成用户已确认决策。记忆随会话保存并优先注入后续请求，不受历史消息压缩影响；切换模型后仍可继续使用。标题栏出现脑形图标表示当前会话已建立项目记忆，悬停或点击可查看概况与分类数量。

临时脚本保存在工作区的 `.gllm/runs` 中，用于后续审计。该目录不会作为用户产物展示，也不会参与目录搜索。

## 安全边界

- 模型只能使用工作区相对路径。
- 主进程同时校验规范路径与真实路径，阻止 `..`、绝对路径和符号链接越界。
- 当前不提供删除工具。
- 当前不提供任意 Shell、系统命令、依赖安装或网络访问。
- 临时脚本在独立进程和受限 VM 中执行，不暴露 `process`、`require`、`fetch` 或供应商 API Key。
- 子进程启用 Node 文件权限，只允许读写当前授权工作区；运行超过 30 秒会被终止。
- 临时脚本单次代码最多 120 KB，文本文件最多读写 10 MB，二进制文件最多读写 25 MB。
- 写文件单次最多 1 MB，读文本单文件最多 500 KiB。
- 搜索跳过 `.git`、`.gllm`、`node_modules`、`dist` 和 `out`。
- 工具循环最多 14 轮，每轮最多执行 6 个工具调用。
- API Key 只由主进程从现有供应商配置使用，不发送到 Renderer 或工具结果。

## macOS 兼容

文件访问使用 Node.js 跨平台 API 和 Electron 原生目录选择器。临时脚本复用 Electron 自带的 Node.js 运行时，不依赖用户预装 Node.js。图片、PDF 处理使用现有 `@napi-rs/canvas`、PDF.js 与 `pdf-lib`，不调用 Windows 命令或外部系统二进制。相同执行路径适用于 Windows、macOS 和 Linux。
